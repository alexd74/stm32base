From scmRTOS

General

This page describes some implementation details of MSP430 port for msp-gcc compiler.

TISRW_SS usage note

As noted in MSP430 Ports section, it is possible to use separate stack while executing interrupt handlers. This should reduse RAM usage but increases interrupt execution time.

But with msp-gcc this smart idea fails because compiler reserves some space on stack before calling TISRW_SS constructor. Then TISRW_SS constructor switches to another stack, and after this any attempt to use reserved space will crash.

If you however want to use TISRW_SS, your should reserve some space on main stack. To do so, define 

#define TISRW_SS_STACK_PAD  (4)

in file scmRTOS_TARGET_CFG.h. This defines the amount of reserved main stack space to avoid this problem.

Value 4 is enough for interrupt handlers like 

OS_INTERRUPT interrupt(UART0RX_VECTOR) usart0_rx(void)
{
    OS::TISRW_SS ISRW;
    Uart1.rx_interrupt_proc();
}

In other cases you should check assembler listing and correct this value appropriately.

For more info about configuring and building MSP430 GCC port read MSP430 GCC Release page.
