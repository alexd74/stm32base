AVR GCC Port

From scmRTOS

General

This page describes some implementation details of AVR port for GCC compiler.

avr-gcc port is intended to be compiled by avr-gcc versions 4.x but v3.4.6 is tested too. Versions before 3.0 are not supported.

Complete list of versions of avr-gcc which current release is tested with see on AVR GCC Release page.

Stack

GCC port for AVR uses single common stack for data and program flow control (call/return). Data RAM filled by variables from bottom while stack pointer initialized with top memory address by default. So, the "main" stack already has maximum possible size. The stack used by startup code, static objects constructors and main() function but unused after OS::Run() called.

As noted in AVR Ports section, it is possible to use separate stack while executing interrupt handlers. AVR GCC port uses this "main" stack for interrupt routines with TISRW_SS wrapper by assigning stack pointer to address of linker-generated label __stack.

For more info about configuring and building AVR GCC port read AVR GCC Release page.
