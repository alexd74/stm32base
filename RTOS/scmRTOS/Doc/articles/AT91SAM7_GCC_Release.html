General

This page describes AT91SAM7 scmRTOS release for GCC toolchain. Tested on Yagarto 20111119 (GCC 4.6.2) release. Other ARM GCC toolchains should work too, possible with some linker script corrections.

Release consists of the RTOS sources and four sample projects - see Release description for details. Each sample located in separate folder. Structure of sample folder (folders are in brackets '[ ]') is follows: 

 [Config]          - linker scripts and other project config files
 [release]         - executable product[s]
    [dep]          - dependency files
    [lst]          - listings, map files
    [obj]          - object files
 [src]             - project sources
 makefile          - makefile for GNU make

If folder release and/or it's subfolders are not present, they are created during build process.

Structure of the RTOS folder: 

 scmRTOS
   Common            - common RTOS sources
   ARM7              - portable part
   Extensions        - OS extensions


See Distribution section and below about release using manners.

Building

To build sample project just type make on command prompt. Make sure that make utility and arm-gcc toolchain's bin folder are in PATH.

When using some IDE (like Eclipse or Code::Blocks) - just set up IDE to use this makefile. Or, if you need to specify command line options for compiler, assembler and linker, use this settings (1):

Assembler 

 -mcpu=arm7tdmi
 -g
 -I../scmRTOS/Common
 -I../scmRTOS/ARM7

Compiler - c files 

 -mcpu=arm7tdmi
 -Os
 -g
 -std=gnu99
 -ffunction-sections -fdata-sections
 -Wall -Wextra
 -Wimplicit -Wcast-align -Wpointer-arith -Wredundant-decls
 -Wshadow -Wcast-qual -Wnested-externs
 -I../scmRTOS/Common
 -I../scmRTOS/ARM7

Compiler - cpp files 

 -mcpu=arm7tdmi
 -Os
 -g
 -fno-exceptions -fno-rtti
 -ffunction-sections -fdata-sections
 -funsigned-bitfields -fshort-enums
 -Wall -Wextra
 -Winline
 -Wimplicit -Wcast-align 
 -Wpointer-arith -Wredundant-decls
 -Wshadow -Wcast-qual
 -I../scmRTOS/Common
 -I../scmRTOS/ARM7
 -I../scmRTOS/Extensions/Profiler

You should also add -mthumb-interwork to compiler options if any of project files compiled in thumb mode.

Note that scmRTOS/ARM7/OS_Target_cpp.cpp and any file with interrupt handlers must be compiled in ARM (not thumb) mode

See sample 1-EventFlag for example of mixing ARM and THUMB mode sources.

Linker 

 -mcpu=arm7tdmi
 -nostartfiles
 -Wl,--gc-sections
 -T./Config/scmRTOS_SAM7_rom.ld

[1] you possibly need to edit Config/scmRTOS_SAM7_rom.ld and/or device.h files for AT91SAM7 chip other than AT91SAM7S64.
